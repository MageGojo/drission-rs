## 里程碑 47 · 吐环境指纹回放深化(④)

### 目标
里程碑 31 已覆盖 **canvas(toDataURL)/ webgl / audio** 的录制+Node 回放+双跑验证。本里程碑补齐
进度表「后续可增强 #1」列出的四个高强度指纹点,使补环境 `env.js` 在 Node 侧能忠实回放:

1. **字体枚举**(font enumeration):`measureText` 宽度法探测已装字体。
2. **像素级 canvas**(`getImageData`):读取像素并哈希(很多指纹库读 `getImageData` 而非 `toDataURL`)。
3. **WebRTC**:`RTCPeerConnection` 是否可用 + 编解码能力概要(默认 `block_webrtc=true` 时即 `supported:false`)。
4. **`navigator.plugins` / `navigator.mimeTypes`**:插件/ MIME 列表(常被读做指纹)。

### 关键设计:录制==回放「单一配方」(沿用里程碑 31)
- 新增配方函数放进 `assets/fp_recipes.js`(探针采集 seed 与 verify 快照**共用同一份**):
  - `__fpFonts()` → `{ supported, detected:[...], widths:{"<font shorthand>":w}, hash }`
  - `__fpCanvasPixels()` → `{ supported, width, height, hash, data }`(`data` = `getImageData` 字节的 base64,供回放)
  - `__fpRtc()` → `{ supported, codecsHash }`
- 配方只依赖 `document.createElement('canvas').getContext('2d')` / `typeof RTCPeerConnection`:
  - **浏览器**里走真实实现 → 真实指纹(录制)。
  - **Node** 里走 `env.js` 注入的假 DOM(回放):
    - `measureText` 返回**录制的宽度**(按 `ctx.font` 串查表)→ `__fpFonts()` 复算出同样的 detected/hash。
    - `getImageData` 返回**录制的字节**(从 `seed.fingerprint.canvasPixels.data` 还原)→ 同 hash。
    - `RTCPeerConnection` 缺省不存在 → `supported:false`(与默认 `block_webrtc` 浏览器侧一致)。

### 录制(探针 `collectSeed`)
- `seed.fingerprint` 增 `fonts` / `canvasPixels` / `rtc`(均同步配方,不走 audio 的异步路径)。
- `seed.navigator` 增 `plugins` / `mimeTypes`(数组,元素含 name/filename/description、type/suffixes 等)。

### 回放(`env_template.js`)
- `makeCtx2D`:`measureText` 改为**查录制宽度**(`__SEED__.fingerprint.fonts.widths[this.font]`,缺失回退旧估算);
  `getImageData` 改为**还原录制字节**(`canvasPixels.data` base64 → `Uint8ClampedArray`,尺寸匹配时用,否则回退 0)。
- `RTCPeerConnection`:`supported:true` 才 stub 一个最小构造器 + `RTCRtpReceiver.getCapabilities`,否则**保持 undefined**。
- `navigator.plugins/mimeTypes`:`setup` 后把 seed 数组包成**类数组**(`length` + 下标 + `item`/`namedItem`/`refresh`),
  并交叉链接 `mimeTypes[i].enabledPlugin`。

### 验证(`dump_env.rs` 双跑 + 导出工程 `verify.js`)
- `gen_snapshot_js`:新增比对项 `fonts.hash`/`fonts.count`、`canvasPixels.hash`、`rtc.supported`、
  `navigator.pluginsCount`/`navigator.plugins0`/`navigator.mimeTypesCount`(后三者浏览器/Node 实时双读)。
- `fill_recorded_fp`:`fonts.hash`/`canvasPixels.hash`/`rtc.*` 用 seed 录制值(canvas 类指纹 Camoufox 可能逐次加噪,
  比对「录制 vs Node 回放」而非「实时 vs 回放」,与 canvas.dataURL 同策略)。
- 导出工程 `verify.js`:增字体/像素/rtc/plugins 的回放自检项。

### 取舍 / 边界
- **footprint**:`canvasPixels` 用紧凑画布(96×32 = 12288 字节 → base64 ~16KB)控制 `env.js` 体积。
- **WebRTC**:默认 `block_webrtc=true` 下 `supported:false`,回放即「RTCPeerConnection 仍 undefined」——这正是要补的一致性。
- **plugins 类型**:回放是类数组(`length`/下标/`namedItem`)而非真 `PluginArray`;`Object.prototype.toString` 仍为 `[object Object]`,
  对绝大多数 `plugins.length`/`plugins[i].name` 读法已足够(进一步伪装可后续按目标站点叠加)。
- 不引入任何新依赖;占位符替换沿用「恰好 1 次」单测兜底(里程碑 20 教训)。
