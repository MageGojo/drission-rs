# 录制生成代码(Recorder / Codegen)+ 无障碍快照(Accessibility Snapshot)

> 里程碑目标:对标 Playwright `codegen` 与 `accessibility.snapshot` / DP 的"易写"。
> 这是 DP / Playwright 有、本库还没有的**降门槛**能力——让用户**录一遍操作就拿到可运行的
> Rust 代码**,并能**一行拿到页面的语义化(角色+名称)结构**用于断言 / 喂给 LLM。

恢复上下文顺序:先读 `docs/需求.md` → `docs/设计.md` → `docs/进度.md`,再看本文件。

## 一、为什么做这个(动机)

- **录制生成代码**:写自动化脚本最耗时的不是逻辑,是"对着页面找选择器"。Playwright 的 `codegen`
  让你点一遍页面就生成代码,极大降门槛。本库语法对齐 DP,录制产物就该是 **DP 风格选择器的 Rust 代码**,
  复制即跑。
- **无障碍快照**:`tab.ax_snapshot()` 把页面压成 `role "name"` 的语义树。两个高价值场景:
  1. **稳定断言**:按"角色+可见名"断言比 CSS/XPath 更抗改版(对齐 PW 的 a11y 断言)。
  2. **喂 LLM**:语义树比整页 HTML 小一个数量级,是给大模型做"页面理解 / 下一步决策"的理想上下文。

## 二、设计总览(组件化 + 后端无关优先)

沿用本库既有分层:**后端无关核心住 crate 顶层(始终编译),协议相关住各后端模块(feature gate)**。

| 模块 | 位置 | 编译条件 | 职责 |
|---|---|---|---|
| `codegen` | `src/codegen.rs` | 始终 | `RecordedAction` / `RecordedScript` 值类型 + 生成 Rust/JSON(纯函数,可单测) |
| `a11y` | `src/a11y.rs` | 始终 | `AxNode` / `AxTree` 值类型 + 快照 JS 脚本 + CDP 扁平树重建(纯函数,可单测) |
| 录制句柄 | `src/cdp/recorder.rs` | `cdp` | `ChromiumRecorder`:注入录制脚本 + `Runtime.bindingCalled` 收动作 + 导航捕获 |
| 录制脚本 | `src/cdp/assets/recorder.js` | `cdp` | 页面内事件钩子(click/change/keydown)+ 计算 DP 风格选择器 → 回调 binding |
| 无障碍接线 | `src/cdp/a11y.rs` | `cdp` | `tab.ax_tree()`(CDP 原生)/ `tab.ax_snapshot()` / `tab.ax_find()` |
| camoufox 平价 | `src/browser/tab.rs` | `camoufox` | `tab.ax_snapshot()`(复用共享快照 JS,跨后端一致) |

**为什么录制只先做 CDP**:录制依赖 `Runtime.addBinding` + `Runtime.bindingCalled`(页面回调宿主),
这与 `expose_function` / `console()` 同机制,CDP 上成熟可靠;且"标配补齐"既有先例就是 **CDP 专有**
(PDF/HAR/媒体模拟…)。Camoufox 的 Juggler binding 语义不同,留作后续。无障碍**快照**因为是纯 JS
(`run_js`),两后端都给。

## 三、录制 + 代码生成(Recorder / Codegen)

### 3.1 录制的动作模型(`RecordedAction`)

刻意只收**少量、稳定、可一一映射到本库 API** 的动作,产物干净:

| 动作 | 触发(页面事件) | 生成的 Rust |
|---|---|---|
| `Navigate{url}` | 主框架 `Page.frameNavigated`(仅 http/https/file,去重) | `tab.get("url").await?;` |
| `Click{selector,frame}` | `click`(排除文本框/下拉/复选,这些由 change 记录) | `tab.click("sel").await?;` |
| `Fill{selector,text,frame}` | 文本输入 `change`(提交/失焦时,取最终值) | `tab.input("sel", "text").await?;` |
| `Check{selector,checked,frame}` | checkbox/radio `change` | `tab.ele("sel").await?.set_checked(true).await?;` |
| `Select{selector,value,frame}` | `<select>` `change` | `tab.ele("sel").await?.select_value("v").await?;` |
| `Press{selector,key,frame}` | 文本框上回车 `keydown=Enter` | `tab.ele("sel").await?.input_keys(&[KeyInput::key(Keys::ENTER)]).await?;` |
| `Hover{selector,frame}` | `mouseover`(可悬停元素 + 停留 ~250ms 防抖) | `tab.ele("sel").await?.hover().await?;` |
| `Drag{from,to,frame}` | HTML5 DnD(`dragstart`→`drop`)或指针按住移动 | `tab.actions().move_to_ele(&from,0.2).hold().move_to_ele(&to,0.4).release().perform().await?;` |
| `NewTab` | 本标签打开弹窗(`Target.targetCreated`,openerId 命中) | `let tab_2 = tab.wait().new_tab(None).await?.expect(..);`(后续动作切到 `tab_2`) |

> - 文本输入按 `change`(失焦/回车提交)记录最终值,与 Playwright codegen 同策略——避免逐字符噪声。
> - **iframe**:元素动作带 `frame`(所在 iframe 选择器,同源经 `window.frameElement` 计算);生成
>   `tab.get_frame("FRAME").await?.ele("SEL").await?.<op>()`。`frame=None` = 顶层文档。
> - **多标签**:`NewTab` 让 codegen 把后续动作切到新标签变量 `tab_2`/`tab_3`…(线性流程;录制器自动附着
>   本标签打开的弹窗、注入录制脚本)。

### 3.2 选择器策略(页面内 JS 计算,DP 风格)

优先级(越靠前越稳):

1. **`#id`** —— 元素有 `id` 且全页唯一。
2. **`@name:val`** —— 表单元素有 `name` 且该 `tag[name=..]` 唯一(本库解析为 XPath 属性匹配)。
3. **`css:<path>`** —— 自底向上构造 CSS 路径(`tag:nth-of-type(k)`,遇到唯一 id 祖先即截断)。

全部是本库 `locator` 支持的写法(`#`/`@attr:`/`css:`),生成的代码 `tab.ele(...)` 直接能用。

### 3.3 录制机制(`ChromiumRecorder`)

- `tab.recorder()` 拿句柄;`start()`(**导航前调用**):
  - `Runtime.enable` + `Runtime.addBinding("__drission_record")`(收页面动作)。
  - `Page.addScriptToEvaluateOnNewDocument(RECORDER_JS)` 让每次导航后录制脚本自动就位;当前文档也即时注入。
  - 后台泵订阅事件:`Runtime.bindingCalled`(动作)+ `Page.frameNavigated`(主框架导航)→ 解析进缓冲。
- `actions()` 看实时已录动作;`stop() -> RecordedScript` 收尾;`Drop` 自动 abort 后台泵(同既有句柄)。
- **反检测取舍**:同 `console()`/`expose_function`,`start()` 开 `Runtime.enable`(经典探测点)。录制是
  **开发期**行为,不在过盾生产链路用,故可接受;文档明确标注。

### 3.4 产物(`RecordedScript`)

- `to_rust()` → 完整可运行程序(`use drission::prelude::*;` + `#[tokio::main]` + 启动/动作/`quit`)。
- `to_rust_body()` → 仅动作语句(嵌进已有代码用)。
- `to_json()` → 动作数组(便于二次加工 / 跨语言 codegen 的中间表示)。
- 字符串字面量统一用 `serde_json` 转义,安全嵌入 Rust 源码。

## 四、无障碍快照(Accessibility)

### 4.1 值类型(`AxNode` / `AxTree`,后端无关)

```text
AxNode { role: String, name: String, value: Option<String>,
         description: Option<String>, properties: BTreeMap<String,String>,
         children: Vec<AxNode> }
AxTree { root: AxNode }   // to_outline / to_json / find_by_role / find_by_name / count
```

- `to_outline()`:缩进文本,每行 `role "name"`(+ 关键属性),给人读 / 喂 LLM 最省 token。
- `find_by_role(role)` / `find_by_name(substr)`:在树上检索,做断言或定位。
- `serde` 派生 → `to_json()` 直接导出。

### 4.2 两条获取路径

1. **`tab.ax_tree()`(CDP 原生,最准)**:`Accessibility.getFullAXTree` 取**扁平**节点数组
   (`role.value`/`name.value`/`value.value`/`properties[]`/`childIds[]`/`ignored`),按 `childIds`
   **重建成树**(跳过 `ignored`)。角色/名称来自 Chrome 的无障碍引擎,权威。仅 CDP。
   - 重建逻辑 `build_from_cdp(flat) -> AxTree` 放 `a11y.rs`(纯函数,喂样本 JSON 可单测)。
2. **`tab.ax_snapshot()`(DOM 派生,跨后端)**:注入 `AX_SNAPSHOT_JS` 在页面里按 ARIA 规则
   (显式 `role` 或按标签推断隐式角色 + 可见名 aria-label/alt/placeholder/text)算一棵近似语义树,
   `run_js` 取回 JSON,`build_from_snapshot(json)` 转 `AxTree`。CDP 与 Camoufox 都可用、结果一致。

> 取舍:`ax_tree()` 准但仅 CDP 且需 `Accessibility` 域;`ax_snapshot()` 近似但零协议依赖、跨后端、
> 可在任何 `run_js` 能跑的地方用。两者都产 `AxTree`,接口一致。

## 六、验证

- **纯函数单测**(不开浏览器,CI 友好):
  - codegen:每种动作 → Rust 行映射;`to_json` 形状;字面量转义。
  - a11y:`build_from_cdp` 扁平→树重建(含 ignored 跳过 / childIds 关联);`to_outline` 缩进;
    `find_by_role`/`find_by_name`;`build_from_snapshot` 解析。
- **端到端示例** `examples/cdp/cdp_recorder.rs`(进程内 HTTP 服务,完全离线):
  起录制 → 程序化驱动(可信点击 / 输入 / 勾选 / 下拉 / 回车,触发页面事件被录制脚本捕获)→ `stop()`
  打印生成的 Rust;再 `ax_snapshot()` / `ax_tree()` 打印语义大纲并断言关键节点。全部通过打印
  `ALL CHECKS PASSED`。
