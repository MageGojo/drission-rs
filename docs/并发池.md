# 高并发规模化采集 设计(BrowserPool / 代理池 / 指纹池 / 断点续抓)

> 目标(对标需求文档头号目标"高性能 / 并发、大规模采集"):把"单浏览器 + 手动管标签"
> 升级为"**浏览器 worker 池 + 任务队列 + 并发上限 + 健康自愈 + 每任务轮换代理/指纹 + 失败重试 + 断点续抓**"。
> **不破坏**现有单浏览器 API(`Browser`/`Tab` 全部保留),池是其上的编排层。

## 1. 关键约束(决定架构的两层指纹)

读 `apply_context_overrides`(`tab.rs`)与 `build_camou_config`(`options.rs`)后确认:

| 维度 | 作用域 | 下发方式 | 能否 per-Tab 轮换 |
|---|---|---|---|
| 代理 proxy | **context 级** | `Browser.setContextProxy{browserContextId}` | ✅ 同进程内每标签可不同 |
| UA / locale / timezone / platform / geolocation | **context 级** | `Browser.set*Override{browserContextId}` | ✅ 同进程内每标签可不同 |
| viewport / bypassCSP / ignoreHTTPSErrors | **context 级** | `Browser.set*{browserContextId}` | ✅ |
| canvas / webgl / 字体 / screen 深指纹、humanize | **进程级** | 启动前 `CAMOU_CONFIG_*` 环境变量 | ❌ 只能 per-Browser 进程 |

⇒ **两层轮换模型**:
- **轻量轮换(默认,高并发)**:代理 + UA/locale/tz/platform/geo/viewport,落在 **per-context**——
  一个浏览器进程内开多个独立 context,每个任务一份,**便宜、并发高**。
- **深度轮换(可选,强隔离)**:canvas/webgl/screen/humanize 深指纹只能 **per-Browser 进程**——
  通过给池里**不同 worker 配不同 `BrowserOptions`(含 `camou_config` seed)** 实现。

> 结论:`BrowserPool` = 一组 Browser worker 进程;每个 worker 有自己的深指纹基线;
> 任务在 worker 内开 context,套上**轻量轮换**的代理 + 指纹。两层都覆盖,既高并发又能强隔离。

## 2. 组件拆分(`src/pool/`,组件化)

```
src/pool/
  mod.rs            BrowserPool / PoolOptions / RetryPolicy 顶层编排 + 导出
  rotate.rs         RotateStrategy(RoundRobin/Random/Sticky)+ std-only xorshift RNG
  proxy_pool.rs     ProxyPool:代理轮换
  fingerprint.rs    FingerprintProfile + FingerprintPool:轻量指纹轮换 + 预设
  checkpoint.rs     Checkpoint:断点续抓(文件持久化已完成 key)
```

外加对底层的一处**非破坏扩展**(`browser/`):
- `tab.rs`:`ContextOverride` 结构 + `Tab::open_with(conn, opts, &ContextOverride)`;
  `apply_context_overrides` 改为接受"基线 opts + 覆盖",覆盖项优先(原 `Tab::open` 等价于空覆盖)。
- `mod.rs`(Browser):`Browser::new_tab_with(&ContextOverride)`——基于自身 opts 克隆 + 覆盖后开 Tab。

## 3. 轮换策略(`rotate.rs`)

```rust
pub enum RotateStrategy { RoundRobin, Random, Sticky }
```
- `RoundRobin`:原子自增取模,均匀。
- `Random`:std-only xorshift(seed 取 nanos + 原子扰动,**不引 `rand`**,沿用项目既有 std-only 习惯)。
- `Sticky`:按外部 key(如域名 / 任务 id)哈希定位,**同 key 稳定拿到同一个**(同账号固定出口)。

`Rotator<T>` 通用:`pick(&self, key: Option<&str>) -> Option<&T>`。ProxyPool / FingerprintPool 各持一个。

## 4. ProxyPool(`proxy_pool.rs`)

```rust
let proxies = ProxyPool::new(vec![Proxy::new("socks5://h:1080"), ...])
    .strategy(RotateStrategy::RoundRobin);
let p: Option<Proxy> = proxies.next();          // 轮询/随机
let p = proxies.for_key("user-42");             // 粘性:同 key 同出口
```
- 空池 → `next()` 返回 `None`(任务用 worker 基线代理 / 无代理)。
- 线程安全(`Arc` + 原子游标),可在多任务并发 `next()`。

## 5. FingerprintPool(`fingerprint.rs`)

只管**轻量层**(per-context 能下发的字段);深指纹通过 worker 的 `BrowserOptions` 配。

```rust
pub struct FingerprintProfile {        // 都是 context 级可覆盖项
    pub user_agent: Option<String>,
    pub locale: Option<String>,
    pub timezone_id: Option<String>,
    pub platform: Option<String>,
    pub os: Option<OsType>,
    pub geolocation: Option<Geolocation>,
    pub window_size: Option<(u32,u32)>,
}
```
- `FingerprintPool::presets()`:内置几组常见且自洽的桌面组合(Win/Mac × 语言/时区),开箱即用。
- `FingerprintPool::new(vec![...])` + `.strategy(..)`:自定义。
- `next()` / `for_key(k)`:轮换;空池返回 `None`(用 worker 基线)。

> 不伪造 canvas/webgl per-context(协议层做不到,见 §1);需要深度差异就给不同 worker 配
> `BrowserOptions.camou_config`(逃生舱已支持任意 `CAMOU_CONFIG_*` 字段)。

## 6. BrowserPool(`mod.rs`,核心编排)

```rust
pub struct PoolOptions {
    pub size: usize,                  // worker(浏览器进程)数,默认 4
    pub tabs_per_worker: usize,       // 每 worker 并发 context 数,默认 1(强隔离)
    pub base_options: BrowserOptions, // 各 worker 启动基线(可被 worker_options 覆盖)
    pub worker_options: Vec<BrowserOptions>, // 可选:逐 worker 不同深指纹基线
    pub proxies: Option<ProxyPool>,
    pub fingerprints: Option<FingerprintPool>,
    pub retry: RetryPolicy,
    pub close_tab_after_task: bool,   // 任务后关 context(默认 true,防泄漏 + 强隔离)
}

pub struct RetryPolicy { pub max_retries: u32, pub backoff: Duration, pub backoff_factor: f64 }
```

并发模型:
- 启动时并行 `Browser::launch` 起 `size` 个 worker;总并发槽 = `size * tabs_per_worker`,用 `Semaphore` 限流。
- `run(task)`:取一个并发许可 → 选 worker(轮询)→ 取代理/指纹拼 `ContextOverride` →
  `worker.new_tab_with(ov)` → 跑 `task(tab)` → (默认)关 tab → 释放许可。失败按 `RetryPolicy` 重试。
- `map(items, task)`:对每个 item 并发跑(`buffer_unordered` 风格,受 Semaphore 限流),返回 `Vec<(item, Result<T>)>`,
  顺序与输入一致。
- `map_resumable(items, key_fn, ckpt, task)`:同上,但跳过 checkpoint 中已完成的 key、完成即落盘(断点续抓)。

健康自愈:
- worker 持 `Browser` + `healthy: AtomicBool`。任务因 `Error::Transport`(连接断/子进程退)失败 →
  标记该 worker 不健康;下次取到不健康 worker 时**惰性重建**(`Browser::launch` 新进程顶替),再重试。
- `shutdown()`:并行 `quit()` 所有 worker。`Drop` 兜底沿用 `Browser` 自身的 Drop(杀进程 + 删临时 profile)。

签名(对象安全友好,任务用闭包):
```rust
impl BrowserPool {
    pub async fn launch(opts: PoolOptions) -> Result<Self>;
    pub async fn run<F, Fut, T>(&self, task: F) -> Result<T>
        where F: Fn(Tab) -> Fut + Send + Sync, Fut: Future<Output = Result<T>> + Send, T: Send;
    pub async fn map<I, F, Fut, T>(&self, items: Vec<I>, task: F) -> Vec<(I, Result<T>)> ...;
    pub async fn map_resumable<...>(&self, items, key_fn, ckpt: &Checkpoint, task) -> ...;
    pub fn concurrency(&self) -> usize;
    pub async fn shutdown(self) -> Result<()>;
}
```
> `task: Fn(Tab)->Fut`(非 `FnOnce`),因为重试要多次调用。`Tab` 是 `Clone`+`Send`,可移进 future。

## 7. Checkpoint 断点续抓(`checkpoint.rs`)

最简可靠:**JSONL 追加**已完成 key(进程崩溃也不丢已写部分)。

```rust
let ckpt = Checkpoint::load("dump/progress.jsonl").await?;   // 读回已完成 key 集合
if !ckpt.is_done(&key) { /* 干活 */ ckpt.mark_done(&key, Some(value)).await?; }
ckpt.done_count();
```
- `load`:逐行读 `{ "key": "...", "value": ... }`,建 `HashSet<key>`;文件不存在 = 空。
- `mark_done`:`OpenOptions::append` 追加一行(`Mutex` 串行化写);幂等(重复 mark 同 key 不重复计)。
- 与 `map_resumable` 配合:**重跑只补未完成**,适合大规模分页 / id 列表采集中断后续作。

## 8. 取舍 / 边界

- **轻量 vs 深度**:默认只轮换轻量层(够绝大多数反 IP 关联 / 多账号场景)。深指纹差异成本高(多进程),
  按需用 `worker_options` 给少数 worker 配不同 `camou_config`。
- **`tabs_per_worker`**:默认 1 = 一个任务独占一个进程上下文,最强隔离;调大提并发但同进程共享深指纹。
- **代理验活不在本期**:`ProxyPool` 只做轮换;失效代理由任务失败 + 重试(换下一个)兜底。
- **不引新依赖**:RNG 用 std-only xorshift;checkpoint 用 `serde_json` + `tokio::fs`(已在依赖里)。
- **离线自验证**:示例 `pool_crawl` 用 `data:`/`file://` 页跑通**并发 + cookie 隔离 + 指纹轮换(读 `navigator`)
  + 重试 + 续抓**,不依赖外网(沿用项目其它示例的离线策略)。代理真实出口需自备代理,单测覆盖轮换逻辑。

## 10. 实测发现(踩坑记录)

- **per-context 覆盖哪些真生效(`pool_crawl` 实测)**:同一进程内每个 BrowserContext 套不同覆盖后,
  - ✅ **时区**:`Browser.setTimezoneOverride` → `Intl.DateTimeFormat().resolvedOptions().timeZone`
    **逐 context 生效**,6 任务读到的时区 distinct=5、且**顺序恰为预设轮询序**(证明 per-context 覆盖
    正确、且并发下"任务↔指纹"对应不串台)。
  - ✅ **视口**:`Browser.setDefaultViewport` → `window.innerWidth` 逐 context 生效(distinct=4)。
  - ❌ **`navigator.language`**:`Browser.setLocaleOverride` **改不动** `navigator.language`(6 任务全 `en-US`,
    distinct=1)。原因:Camoufox 的 `navigator.language` 由**进程级**指纹层(`CAMOU_CONFIG`)固定,
    Juggler 的 per-context locale 覆盖只影响 **Accept-Language 请求头**(服务端可见、对反关联仍有用),
    不影响页面里读到的 `navigator.language`。
  - ⇒ **结论**:轻量指纹池的 locale 用于"服务端 Accept-Language 差异 + 时区联动";要让**页面 JS 读到的
    `navigator.language` 每身份不同**,必须给不同 worker 配进程级 `BrowserOptions`(`camou_config` 设
    `navigator.language` 等)。`FingerprintPool::presets()` 故意只承诺"时区/视口 per-context + locale 走
    Accept-Language",不谎称改 `navigator.language`。
- **重试与自愈**:`run` 对任意 `Err` 重试(退避);仅 `Error::Transport`(连接断/子进程退)额外置 worker
  `healthy=false`,下次取用时惰性 `Browser::launch` 重建——进程级故障自愈,普通任务错误只重试不重建。
- **断点续抓幂等**:`Checkpoint::mark_done` 先写盘再入集合(写失败则不算完成、下次重试);`map_resumable`
  开跑前过滤已完成 key,**重跑只补未完成**(实测首跑 done=4 → 续跑同列表+2 新项只补 2 → done=6)。
- **`map` 保序**:内部 `buffer_unordered(concurrency)` 并发跑、带原始下标,收集后按下标还原,返回顺序 == 输入顺序。
- **无新依赖**:RNG 用 std-only SplitMix64(原子计数,无锁);checkpoint 用 `serde_json` + `tokio::fs`(均已在依赖)。
