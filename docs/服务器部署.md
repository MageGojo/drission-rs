# 在「没有界面(无 UI)的 Linux 服务器」上部署 drission

> 一句话:**你的程序(Rust 二进制)不需要界面;它驱动的「浏览器」也不需要显示器——开无头(headless)即可。**
> 服务器没桌面 / 没显示器 / 没 X,完全能跑;真正要解决的只是「让浏览器及其系统依赖到位 + 容器/root 下别崩」。本文给出两条落地路线。

---

## 0. 先分清两层(很多人卡在这)

| 层 | 是什么 | 需要界面吗 | 怎么解决 |
|---|---|---|---|
| **① 你的程序** | 用 drission 编出来的 Rust 二进制 | **不需要** | musl 静态一份跑遍所有发行版(见 [`scripts/linux-musl-build.sh`](../scripts/linux-musl-build.sh)) |
| **② 浏览器** | 库驱动的 Chrome / Chromium | **不需要**(开无头) | 装浏览器 + 它的系统 `.so` 依赖;容器/root 下加 `--no-sandbox`(库已自动) |

**结论**:musl 静态只解决了 ①;"能在服务器跑"还要把 ② 备齐。本文就是讲 ②。

---

## 方案一(推荐):Docker 一键 —— 目标服务器零配置

镜像里已装好 **Chrome + 全部 `.so` 依赖 + 中日韩字体**,你的程序无头驱动它。交付方只需 `docker run`,**服务器上什么都不用装**。这就是 Linux 版的「零环境交付」。

```bash
# 1) 构建(默认编 cdp_demo 示例;换自己的示例/feature 见下)
docker build -t drission .

# 2) 跑(--shm-size 调大更稳;不调也行,库已自动 --disable-dev-shm-usage 兜底)
docker run --rm --shm-size=1g drission
# 预期输出:title/url/1+2/h1/截图字节数 …… ==== CDP demo 完成 ====
```

换成你自己的示例或 feature 组合:

```bash
docker build --build-arg EXAMPLE=cdp_fetch --build-arg FEATURES=cdp,ocr -t drission .
```

出 **x86_64 + ARM64** 两份(覆盖鲲鹏 / 倚天 / 飞腾等信创 ARM 服务器):

```bash
docker buildx build --platform linux/amd64,linux/arm64 -t <you>/drission --push .
```

> - **amd64** 镜像装**真·Google Chrome**(过盾最佳);**arm64** 无官方 Linux Chrome,自动改用发行版 `chromium`。
> - **把本库用进你自己的项目**:把 `Dockerfile` 第二阶段当基础镜像(已带 Chrome + 字体 + `CHROME_BIN`),只 `COPY` 进你自己的二进制、改 `ENTRYPOINT` 即可。
> - **缓存模型/浏览器避免重复下载**:挂卷 `-v drission-cache:/home/app/.cache`(OCR 模型、Chrome for Testing 都缓存在 `~/.cache/drission`)。

`docker-compose.yml` 示例:

```yaml
services:
  crawler:
    build: .
    shm_size: "1gb"
    volumes:
      - drission-cache:/home/app/.cache
    # environment:
    #   - DRISSION_OCR_MODEL=/home/app/.cache/drission/ocr/common.onnx
volumes:
  drission-cache:
```

## 方案二:裸机 musl 静态二进制 + 装系统依赖

不想用 Docker、想直接把二进制 scp 到服务器跑,就用这条。

### 1) 出静态二进制(在你的开发机 / CI 上)

```bash
scripts/linux-musl-build.sh                       # 默认 x86_64 + aarch64 两份,自动验证「statically linked」
TARGETS="x86_64-unknown-linux-musl" scripts/linux-musl-build.sh   # 只要一份
```

产物在 `target/<triple>/release/`,**完全静态、不依赖 glibc**,CentOS7 / Alinux / Ubuntu / Alpine 通吃。scp 到服务器即可。

### 2) 在服务器上把「浏览器 + 它的系统库 + 字体」装好

> 静态二进制带的是**你的程序**;浏览器(Chrome/Chromium)是独立程序,它自己**动态依赖**一堆系统 `.so`(`libnss3`/`libgbm`/`libasound2`/`libatk-1.0`/`libxkbcommon`/`libX11`…)+ 字体。这些 musl 静态**带不进去**,得在目标机装。

**Debian / Ubuntu**(最省事:直接装 Chrome,自动拉齐所有依赖):

```bash
# 真·Google Chrome(推荐,过盾最佳)
sudo apt-get update && sudo apt-get install -y wget gnupg
wget -qO- https://dl.google.com/linux/linux_signing_key.pub | sudo gpg --dearmor -o /usr/share/keyrings/google-chrome.gpg
echo "deb [arch=amd64 signed-by=/usr/share/keyrings/google-chrome.gpg] http://dl.google.com/linux/chrome/deb/ stable main" | sudo tee /etc/apt/sources.list.d/google-chrome.list
sudo apt-get update && sudo apt-get install -y google-chrome-stable fonts-noto-cjk fonts-liberation
export CHROME_BIN=/usr/bin/google-chrome-stable
# 或装发行版 chromium:sudo apt-get install -y chromium fonts-noto-cjk
```

**CentOS7 / Alibaba Cloud Linux / RHEL 系**:

```bash
sudo yum install -y chromium google-noto-sans-cjk-fonts liberation-fonts || \
  sudo dnf install -y chromium google-noto-sans-cjk-fonts liberation-fonts
export CHROME_BIN=$(command -v chromium-browser || command -v chromium)
```

**Alpine**:

```bash
apk add --no-cache chromium nss freetype harfbuzz ttf-freefont font-noto-cjk
export CHROME_BIN=/usr/bin/chromium-browser
```

### 3) 跑

```bash
CHROME_BIN=/usr/bin/google-chrome-stable ./your_binary
```

> 不想装系统 Chrome?库的 `ensure_chrome()` 会**自动下载 Chrome for Testing** 到 `~/.cache/drission`。但注意:**下载的 Chrome 仍是动态链接的**,上面那串系统 `.so` 依赖**照样得装**(下载的只是浏览器本体,不含系统库)。所以最小化镜像/裸机仍需先 `apt/yum/apk` 装依赖。

## 关键:代码里开无头 + 库自动兜底

```rust
use drission::prelude::*;

#[tokio::main]
async fn main() -> drission::Result<()> {
    // 服务器无显示器 → 开无头即可,不需要任何桌面/X。
    let browser = ChromiumBrowser::launch(ChromiumOptions::new().headless(true)).await?;
    let tab = browser.new_tab(Some("about:blank")).await?;
    tab.get("https://example.com").await?;
    println!("{:?}", tab.title().await?);
    browser.quit().await?;
    Ok(())
}
```

**库已经在 Linux 自动做的事**(`src/cdp/browser.rs`,无需你手动加):

- **`--no-sandbox`**:以 root 运行(Docker / 多数云主机默认 root)时 user-namespace sandbox 起不来(`No usable sandbox!`),库自动旁路。已配好非 root 沙箱可设 `DRISSION_NO_SANDBOX=0` 关掉。
- **`--disable-dev-shm-usage`**:容器 `/dev/shm` 默认仅 64MB,渲染进程写爆会 `Tab crashed`,库自动改用临时文件。

## 反检测 / 过盾在服务器上的注意

- 本库无头默认就**精准对齐有头 Chrome 指纹**(里程碑 52/53:`--headless=new` + 自动补 GPU/屏幕),无头也能过 Cloudflare Turnstile。**要过盾就用完整 Chrome**(本方案默认即是)。
- **不要图轻量换 `chrome-headless-shell`**:它是**老无头**重新打包,指纹破绽更多、更容易被风控识破。纯采集(不过盾)可以用它换速度/体积,但**过盾场景别用**。
- **代理**:`ChromiumOptions::new().proxy("http://host:port")`(或 `socks5://`),服务器出口被风控时按需挂。

## 兜底:个别站点对「纯无头」风控极严 → 用 Xvfb 跑「有头」

极少数站点即使指纹干净也专门识别无头。此时用**虚拟显示器**跑**有头**(`headless(false)`)——服务器仍无物理显示器,但浏览器以为自己有真实显示:

```bash
sudo apt-get install -y xvfb           # Debian/Ubuntu
xvfb-run -a ./your_binary              # 程序里设 .headless(false)
```

Docker 里同理:`RUN apt-get install -y xvfb`,`ENTRYPOINT ["xvfb-run","-a","/usr/local/bin/drission-app"]`。

## 常见崩溃 / 报错速查

| 现象 | 原因 | 解决 |
|---|---|---|
| `No usable sandbox! / Failed to move to new namespace` | root/容器无 sandbox | 库已自动 `--no-sandbox`;若自定义启动确认没把它去掉 |
| `Tab crashed` / 页面白屏崩 | `/dev/shm` 太小(容器 64MB) | 库已自动 `--disable-dev-shm-usage`;或 `docker run --shm-size=1g` |
| `error while loading shared libraries: libnss3.so …` | 缺浏览器系统 `.so` 依赖 | 按「方案二·第 2 步」装依赖(或直接装 Chrome 自动拉齐) |
| `Failed to connect / cannot open display` | 跑了**有头**但没显示器 | 改 `.headless(true)`,或 `xvfb-run` |
| 页面中文全是 `▯▯▯` 方块 | 缺中日韩字体 | 装 `fonts-noto-cjk`(本镜像已带) |
| `GLIBC_2.xx not found` | 用了 **gnu** 动态二进制拷到老系统 | 改用 **musl 静态**(`scripts/linux-musl-build.sh`)或用 Docker |
| 找不到浏览器 | 没装 / 路径不对 | 设 `CHROME_BIN=/path/to/chrome`,或让 `ensure_chrome()` 自动下载 |

## 环境变量速查

| 变量 | 作用 |
|---|---|
| `CHROME_BIN` / `DRISSION_CHROME` | 指定浏览器可执行文件(定位优先级最高) |
| `DRISSION_NO_SANDBOX=0` | 关掉 Linux 自动 `--no-sandbox`(已配好非 root 沙箱时) |
| `DRISSION_HEADLESS_GPU` | 无头 GPU 策略:`0/software` 软渲染 · `1/hardware` 硬件 |
| `DRISSION_OCR_MODEL` / `DRISSION_OCR_MODEL_URL` | OCR 模型本地路径 / 自定义下载源(`ocr` feature) |

---

## 一句话总结

> **musl 静态(库分发)+ 无头(无需显示器)+ 库自动 `--no-sandbox`/`--disable-dev-shm-usage`(root/容器不崩)+ 装好浏览器及其系统依赖与字体(Docker 已打包好)= 你的库在没 UI 的国内服务器上开箱即跑。**
> 最省心:**用 `Dockerfile` 出镜像,`docker run` 即用。**
